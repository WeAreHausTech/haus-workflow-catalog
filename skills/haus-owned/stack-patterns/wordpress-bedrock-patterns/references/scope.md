# Scope

## In-scope files and dirs

- `config/application.php` — WP config constants, env loading via `Dotenv`
- `config/environments/` — per-environment overrides (`development.php`, `staging.php`, `production.php`)
- `composer.json`, `composer.lock` — PHP dependency management (WP core, plugins, mu-plugins via Packagist/WPackagist)
- `web/app/` — WordPress web root: `themes/`, `plugins/`, `mu-plugins/`, `uploads/`
- `web/wp/` — WordPress core (Composer-managed, do not edit)
- `web/index.php`, `web/wp-config.php` — web entry points (minimal; generated by Bedrock)
- `.env`, `.env.example` — environment variable files (`DB_*`, `WP_HOME`, `AUTH_KEY`, etc.)
- `Trellis/` (if present) — Ansible provisioning for Bedrock deployment

## Stack boundaries

- Bedrock: env-based config, Composer-managed WP core and plugins, `web/` vs `app/` separation
- Not in scope: WordPress hook/template logic inside themes/plugins (use wordpress-patterns)
- Not in scope: ACF/Elementor/JetEngine data layer (use wordpress-acf-elementor-jetengine-patterns)
- Not in scope: infrastructure provisioning beyond Trellis config

## Triggers

- Adding or upgrading a plugin via Composer (not WP admin)
- Changing WP version constraint in `composer.json`
- Adding a new constant or config to `config/application.php`
- Adding environment-specific config override in `config/environments/`
- Changing `.env` variables (and updating `.env.example`)
- Adding a mu-plugin or must-use plugin to `web/app/mu-plugins/`
