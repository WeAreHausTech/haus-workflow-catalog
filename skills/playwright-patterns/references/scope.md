# Scope

## In-scope files and dirs

- `playwright.config.ts` — browsers, baseURL, retries, reporter, test dir, timeout config
- `tests/e2e/**/*.spec.ts` — E2E test specs organized by feature/flow
- `tests/e2e/fixtures/**` — custom Playwright fixtures extending base `test`
- `tests/e2e/pages/**` — page object model classes (POM)
- `tests/e2e/helpers/**` — shared utilities (auth setup, API seeding, selectors)
- `.auth/**` — stored auth state files (generated by `storageState`)
- `tests/e2e/global-setup.ts`, `tests/e2e/global-teardown.ts` — global hooks

## Stack boundaries

- Playwright E2E: multi-browser UI flows, form fills, network interception, screenshots
- Auth: use `storageState` for reusable authenticated sessions — avoid logging in every test
- Page objects: encapsulate selectors and interactions; specs only call POM methods
- Not in scope: component unit tests (use testing-library-patterns)
- Not in scope: production app code changes driven purely by test changes

## Triggers

- Adding a new user-facing feature that needs an E2E happy path test
- Fixing a flaky test (selector change, wait strategy, fixture isolation)
- Adding a new browser or viewport configuration
- Changing auth flow tested in E2E (re-record `storageState`)
- Adding network mock or API interception for offline/error path test
- Updating page object selectors after UI refactor
