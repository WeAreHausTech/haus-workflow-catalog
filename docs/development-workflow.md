# Development workflow

## Scope

This repo is a **content catalog** — skills, agents, templates, slash commands, and project-root config files (ESLint, Prettier) — plus the Node.js tooling to validate, sync, and release them. There is no runtime application.

## Common change types

| Change type                       | Primary path(s)                          | Also touch                                                            |
| --------------------------------- | ---------------------------------------- | --------------------------------------------------------------------- |
| Add/edit skill                    | `skills/<source>/<skill-name>/SKILL.md`  | `manifest.json` (version bump + fields)                               |
| Add/edit agent                    | `agents/<source>/<agent>.md`             | `manifest.json`                                                       |
| Add/edit template                 | `templates/<name>.md`                    | `manifest.json`                                                       |
| Add/edit slash command            | `commands/<name>.md`                     | `manifest.json`                                                       |
| Add/edit config (ESLint/Prettier) | `configs/<tool>/`                        | `manifest.json` (`type: config`); distributed via CLI `haus scaffold` |
| Add a new stack tag               | `validation-rules.json#allowedStacks`    | Item files using the tag                                              |
| Edit validation rules             | `validation-rules.json`                  | `scripts/validate.mjs`, `docs/runbook.md`; see ADR-0001 landing order |
| Edit agentic workflow standard    | `templates/agentic-workflow-standard.md` | `cp templates/agentic-workflow-standard.md .claude/WORKFLOW.md`       |
| Edit a script                     | `scripts/*.mjs`                          | `tests/` (add/update tests)                                           |
| Release                           | See [deployment.md](deployment.md)       | —                                                                     |

## Item version bumps

s
Bump the item's `version` in `manifest.json` whenever **any file under that item's `path` changes**:

| Change                                          | Bump            |
| ----------------------------------------------- | --------------- |
| Wording fix, typo, reordering                   | `PATCH` (x.x.1) |
| New guideline, extended scope, new bundled file | `MINOR` (x.1.x) |
| Removed section, breaking scope change, rename  | `MAJOR` (1.x.x) |

CI enforces this on PRs via `scripts/check-item-versions.mjs`.

## Upstream sync (curated items)

Curated items under `skills/<source-slug>/`, `commands/<source-slug>/`, and `agents/<source-slug>/` are copied verbatim from upstream. Do not hand-edit them except for security patches.

```bash
node scripts/sync-upstream.mjs --check    # detect drift
node scripts/sync-upstream.mjs --apply   # apply upstream changes
yarn validate                             # verify after apply
```

Provenance: `sources.yaml`. Automated weekly sync opens PR `chore/upstream-catalog-sync` via `.github/workflows/upstream-sync.yml`. See [runbook.md](runbook.md) for the full reviewer workflow.

## Workflow-doc sync

`templates/agentic-workflow-standard.md` and `.claude/WORKFLOW.md` must be byte-identical — enforced by `checkWorkflowDocSync` in `validate.mjs`.

```bash
cp templates/agentic-workflow-standard.md .claude/WORKFLOW.md
```

Always copy in this direction: template → local copy.

## Quality checks

Pre-commit (Lefthook, runs on `git commit`):

| Check          | Command                 | Scope                              |
| -------------- | ----------------------- | ---------------------------------- |
| Secret scan    | gitleaks + grep         | Staged files                       |
| Catalog valid. | `yarn validate`         | Full manifest + rules              |
| Format         | `yarn prettier --write` | Staged `.json .md .mjs .yml .yaml` |
| Lint           | `yarn eslint`           | Staged `scripts/**` and `tests/**` |

Pre-push (runs on `git push`):

| Check | Command     |
| ----- | ----------- |
| Tests | `yarn test` |

Full CI gate (validate.yml):

```bash
yarn lint
yarn format:check
yarn test
node scripts/validate.mjs ./manifest.json
node scripts/check-item-versions.mjs $BASE_SHA   # PRs only
```

Run `yarn validate && yarn test` locally before pushing to catch failures early.

## Generated artifacts

- `CHANGELOG.md` — generated by `release-it` from Conventional Commits. Do not edit by hand. To rebuild from tags: `yarn changelog:regenerate`.
- `manifest.json#version` — synced by `scripts/sync-version.mjs` during `yarn release`.

## Tests

`tests/` contains a `node:test` suite exercising `scripts/validate.mjs`. Run with `yarn test`.
